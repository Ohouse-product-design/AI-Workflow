Review & Critique

디자인 리뷰 ~ 피드백 반영
P1 Quick Win
Design-critique
비즈니스 목표 정렬, 사용자 중심성, 시각적 위계, 접근성, STACK DS 적용 여부까지 분석. 이미 운영 중인 #des_pd_design_critique 기반.

P2
UX-writing
와이어프레임을 바탕으로 브랜드 가이드에 맞는 카피 작성. 용어 맞춤법, 띄어쓰기 검사, 비문 감지까지 라이팅 품질 향상.
Skill · UX 라이팅 Skill · 맞춤법
P2
Design-share
PO, 기획, 마케팅 대상으로 디자인 초안 공유. 논의 포인트 정리 → 방향성 발산을 위한 피드백 노트 자동 정리.
Skill · 공유 문서 Slack · 알림
P2
Feedback-tracker
PO/기획/마케팅 피드백을 자동 수집 → 프로토타입에 반영. UT 결과 크리틱/분석 → 이터레이션 자동 트래킹.
Agent · 피드백 수집 MCP · Slack

Handoff & QA

Figma Plugin · 상태 체크 Skill · 누락 검증

Skill · ODS 조회 Slack · QnA 봇
P1
Design-handoff-doc

Figma Plugin Skill · 텍스트 검증

Design-QA

Figma · QA 비교 Agent · 자동 검증

Onboarding-kit

솔루션 타입 가이드

각 솔루션 타입의 특성과 적합한 사용 케이스입니다.

Claude Skill
반복적인 단일 작업
슬래시 커맨드(/query, /critique)로 즉시 실행. 프롬프트 + 도구 조합. 개인이 직접 호출.

예: 데이터 조회, UX 라이팅 검수, 스펙 문서 생성
Claude Agent
복잡한 멀티스텝 작업
여러 도구를 자율적으로 조합해 수행. 수집 → 분석 → 정리 등 연쇄 작업. 백그라운드 실행 가능.

예: VOC 수집+분류, 피드백 추적, 경쟁사 리서치
Figma Plugin
디자인 도구 내 자동화
Figma 안에서 직접 실행. 컴포넌트 생성/검증/변환 등 시각적 작업. MCP로 연동 가능.

예: 상태별 UI 생성, 핸드오프 어노테이션, QA 비교
MCP Plugin
외부 시스템 연결
Athena, Redash, Notion, Google Drive 등 외부 데이터 소스 연결. Skill/Agent의 백엔드 역할.

예: 데이터 조회, Slack 메시지 읽기, Notion DB 접근
Slack Bot
팀 채널 기반 자동화
Slack 채널에서 자동 게시/알림. 팀 전체가 수동 호출 없이 혜택. 스케줄 기반 실행.

예: 크리틱 자동 게시, ODS QnA 응답, 공유 알림
