Problem framing

Initiative-prioritizer

Skill · 우선순위 스코어링 MCP · Insight-archive 연계

Data-interpreter

Design & Prototype

DS 기반 생성 → 상태 검토 & Figma 정제
Design system 기반 생성
P0
DS-prototype
PRD + ODS 컴포넌트 레지스트리 → Claude Code로 HTML 프로토타입 생성. DS를 준수한 클릭 가능한 결과물.
Claude Project · ODS 컨텍스트 Skill · Registry
P1
UT-prototyping
화면 흐름 설명 → UT 시나리오 + 태스크 플로우 자동 생성. 사용성 테스트 준비 시간 단축.
Skill · UT 시나리오 생성 문서 · 프로토타입 연결
상태 검토 & Figma 정제
P0
State-edge-simulator
화면 설명 입력 → Loading/Empty/Error 상태 누락 자동 탐지 + 빈값·최대 길이·특수문자 등 엣지 케이스 데이터 자동 생성. 혼자 놓치기 쉬운 케이스를 선제 커버.
Skill · 상태 체크리스트 Skill · 엣지 케이스 생성 기존 도구 · Content Reel
P1
HTML-to-Figma-refiner
AI가 만든 HTML 프로토타입을 Figma로 전환해 UI 디테일(타이포·간격·컬러·상태)을 다듬고 핸드오프용 원본으로 정제. 디자이너 최종 품질 보증 지점.
기존 도구 · Figma Skill · HTML → Figma 변환

Handoff & QA

통합 핸드오프 → 품질 검증 → 카피 & 반응형
P0
Complete-handoff-doc
정책·비즈니스 규칙·엣지 케이스 스펙 + Figma Dev Mode 기반 핸드오프 문서를 하나의 산출물로 통합. 핸드오프 후 질문 왕복 최소화.
Skill · 통합 핸드오프 템플릿 기존 도구 · Figma Dev Mode
P1
Design-QA
디자이너가 Claude Code로 개발된 화면의 CSS/HTML을 직접 수정하고 GitHub에 커밋. PD의 새로운 역할.
Skill · Claude Code 수정 문서 · GitHub 커밋 가이드
P2
UX-writing
브랜드 가이드 기반 버튼 텍스트·에러 메시지·빈 상태 문구 자동 작성 + 맞춤법 검사.
Skill · UX 라이팅 Skill · 맞춤법
P2
Responsive-support
DS-prototype HTML에 CSS 미디어쿼리 포함하여 반응형 자동 해결. 브레이크포인트 가이드 문서화.
문서 · 반응형 가이드 기존 도구 · Figma 플러그인

DESIGN.md

문서 · HTML 스냅샷 Skill · 패턴 검색 기존 도구 · ODS 매핑

솔루션 타입 가이드

각 솔루션 타입의 특성과 적합한 사용 케이스입니다.

Claude Skill
슬래시 커맨드로 즉시 실행
반복적 단일 작업. 프롬프트 템플릿 기반. 개인이 직접 호출.

예: 체크리스트 생성, 데이터 조회, UT 시나리오
Claude Project
시스템 프롬프트 + 문서 컨텍스트
전용 환경을 세팅하여 특정 도메인에 최적화된 응답 생성. ODS 등 대규모 컨텍스트 포함.

예: DS-prototype (ODS 컨텍스트 포함)
문서/템플릿
코드 없이 Notion, MD, 체크리스트로 정리
가장 많고 가장 중요한 기초 작업. 패턴 정리, 가이드 작성, 수동 큐레이션 등.

예: 패턴 정리, 정책 가이드, 온보딩 체크리스트
기존 도구 활용
이미 있는 Figma Plugin/Dev Mode 등 활용
새로 만들지 않고 기존 도구를 그대로 활용. Design Lint, Content Reel 등.

예: Design Lint, Figma Dev Mode, Content Reel
MCP / Slack
외부 시스템 연동
Athena, Redash MCP 또는 Slack Bot을 통해 외부 데이터/워크플로우에 연결.

예: 데이터 조회, ODS QnA 봇, 크리틱 봇
