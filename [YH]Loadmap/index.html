Workflow

6-Step Design Process

업계에서 가장 널리 사용되는 6단계 프로세스입니다. 각 단계는 순차적이지만 필요에 따라 반복(Iteration)됩니다.

01

Discover & Research

사용자와 시장을 이해한다

정량/정성 리서치를 통해 사용자의 행동, 니즈, 페인포인트를 파악하고 시장과 경쟁 환경을 분석합니다. 데이터 기반의 인사이트가 디자인 의사결정의 토대가 됩니다.

User Interview Survey Analytics Review Competitive Analysis Contextual Inquiry Diary Study
02

Define & Synthesize

핵심 문제를 정의한다

리서치 데이터를 종합하여 핵심 문제를 정의합니다. 페르소나, 여정 지도, HMW 질문 등을 통해 팀 전체가 동일한 문제 인식을 갖도록 합니다.

Persona Journey Map HMW Questions Problem Statement Affinity Diagram Jobs-to-be-Done
03

Ideate & Explore

가능성을 발산한다

다양한 솔루션 아이디어를 빠르게 탐색합니다. 양이 질을 만든다는 원칙 아래, 제약 없이 아이디어를 발산한 뒤 가장 유망한 방향을 선택합니다.

Crazy 8's Design Sprint Sketching Storyboard Concept Testing Mind Map
04

Design & Prototype

솔루션을 구체화한다

선택된 아이디어를 와이어프레임, UI 디자인, 인터랙티브 프로토타입으로 발전시킵니다. 디자인 시스템을 활용하여 일관성을 유지하며, 피델리티를 점진적으로 높여갑니다.

Wireframe UI Design Prototype Design System Interaction Design Responsive Design
05

Test & Validate

사용자와 함께 검증한다

프로토타입을 실제 사용자에게 테스트하여 가설을 검증합니다. 유저빌리티 테스트, A/B 테스트 등을 통해 개선점을 발견하고 디자인을 반복 개선합니다.

Usability Testing A/B Testing Heuristic Evaluation Accessibility Audit Guerrilla Testing
06

Deliver & Measure

출시하고 성과를 측정한다

개발팀과 협업하여 디자인을 구현하고, 출시 후 핵심 지표를 모니터링합니다. 정량 데이터를 기반으로 다음 이터레이션을 계획합니다.

Design Spec Dev Handoff QA Review KPI Tracking Retrospective Design Critique

Principles

Design Principles

성공적인 프로덕트 디자이너가 지키는 핵심 원칙들입니다.

User-Centered

모든 디자인 결정은 사용자 리서치와 데이터에 근거합니다. 가정이 아닌 증거 기반으로 의사결정하고, 지속적으로 사용자 피드백을 반영합니다.

Iterate, Don't Perfect

처음부터 완벽한 디자인은 없습니다. 빠르게 만들고, 테스트하고, 학습하는 반복을 통해 점진적으로 개선합니다. 속도가 완벽보다 중요합니다.

Collaborate Cross-functionally

디자인은 혼자 하는 것이 아닙니다. PM, 엔지니어, 리서처, 데이터 분석가와 긴밀하게 협업하여 실현 가능하고 비즈니스 가치가 있는 솔루션을 만듭니다.

Design with Systems

개별 화면이 아닌 시스템으로 디자인합니다. 일관된 디자인 시스템과 패턴을 통해 확장 가능하고 유지보수 가능한 경험을 만듭니다.

Measure Impact

디자인의 성공을 정량적으로 측정합니다. UX 지표(Task Success Rate, SUS, NPS 등)와 비즈니스 지표(Conversion, Retention 등)를 함께 추적합니다.

Accessibility First

모든 사용자가 제품을 사용할 수 있어야 합니다. WCAG 가이드라인을 준수하고, 다양한 디바이스와 환경에서의 접근성을 고려합니다.

AI Automation

AI로 워크플로우 자동화하기

현재 AI 기술 수준에서 프로덕트 디자인 워크플로우의 각 단계를 어떻게 자동화할 수 있는지, 임팩트가 큰 순서대로 정리합니다.

왜 지금 디자인 워크플로우 자동화인가?

2025-26년 현재 LLM, Vision AI, 생성형 AI의 성숙도가 디자인 워크플로우의 핵심 병목을 해결할 수 있는 수준에 도달했습니다. 특히 반복적이고 시간 소모적인 작업에서 AI가 디자이너의 시간을 되찾아주어, 더 높은 가치의 창의적 의사결정에 집중할 수 있게 합니다.

#1 Top

Deliver & Measure — 핸드오프 자동화

Step 06 · Deliver & Measure

디자인-개발 핸드오프는 프로덕트 디자이너 업무 시간의 20-30%를 차지하는 최대 병목입니다. AI가 이 과정을 거의 완전 자동화할 수 있어, 즉시 적용 가능한 가장 높은 ROI를 제공합니다.

  • Figma 디자인 → 프로덕션 코드(React/Swift/Kotlin) 자동 생성 (Cursor, Claude Artifacts)
  • 디자인 변경사항 자동 감지 → Jira 티켓 자동 생성 및 개발자 태깅
  • QA 자동화: 스크린샷 비교로 디자인-구현 차이 자동 검출
  • 출시 후 KPI 대시보드 자동 생성 및 이상 징후 알림
  • 디자인 스펙 문서 자동 작성 (여백, 색상, 인터랙션 동작 명세)

Discover & Research — 리서치 분석 자동화

Step 01 · Discover & Research

정성 리서치 데이터(인터뷰 녹취록, 설문 응답, 리뷰 등)의 분석은 수일이 걸리는 고된 작업입니다. LLM이 대량의 텍스트에서 패턴과 인사이트를 추출하는 데 뛰어난 성능을 보여, 리서치 사이클을 극적으로 단축할 수 있습니다.

  • 유저 인터뷰 녹취록 자동 전사 + 주제별 코딩(Thematic Analysis)
  • 앱스토어 리뷰/CS 로그 대량 분석 → 자동 Pain Point 추출 및 분류
  • 경쟁사 앱 UI 스크린샷 크롤링 → Vision AI로 패턴 분석 리포트 생성
  • 설문 개방형 응답 자동 클러스터링 및 감성 분석
  • 데이터 분석 자동화: 자연어로 질의 → SQL 자동 생성 → 시각화

#3 High

Design & Prototype — UI 생성 가속

Step 04 · Design & Prototype

생성형 AI가 텍스트/스케치에서 UI를 직접 생성할 수 있게 되면서, 디자인 탐색의 속도와 범위가 혁명적으로 변하고 있습니다. 디자이너는 0에서 시작하는 대신 AI가 생성한 안을 큐레이션하고 다듬는 방식으로 전환할 수 있습니다.

  • 텍스트 프롬프트 → UI 와이어프레임/Hi-fi 목업 자동 생성
  • 디자인 시스템 기반 컴포넌트 자동 조합 및 레이아웃 제안
  • 한 화면 디자인 → 다크모드/반응형/다국어 변형 자동 생성
  • 스케치/손그림 → Figma 컴포넌트로 자동 변환
  • 기존 디자인 시스템 학습 → 브랜드 일관성 자동 검증 및 제안

Test & Validate — 테스트 자동화

Step 05 · Test & Validate

유저빌리티 테스트의 모집-실행-분석 사이클을 AI가 대폭 줄여줍니다. 특히 AI 시뮬레이션 기반의 예측적 분석은 실제 테스트 전에 주요 이슈를 사전에 발견할 수 있게 합니다.

  • AI 기반 Heuristic Evaluation 자동 수행 (접근성, 일관성, 가독성)
  • 프로토타입 스크린 분석 → 잠재적 사용성 이슈 예측 리포트
  • 유저 테스트 세션 녹화 자동 분석 (표정, 클릭 패턴, 머뭇거림 감지)
  • A/B 테스트 결과 자동 해석 및 통계적 유의성 판단
  • WCAG 접근성 자동 감사 및 수정 제안
