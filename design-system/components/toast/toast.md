# Toast 사용 규칙

## Variants
| Variant | 용도 | 색상 |
|---|---|---|
| **Info** | 일반 안내 (스크랩, 복사 등) | 다크 (`gray.900`) |
| **Success** | 성공 (저장, 등록 등) | 그린 (`green.50` + `green.900`) |
| **Error** | 에러 (실패, 오류) | 레드 (`red.50` + `red.900`) |
| **Warning** | 경고 (네트워크, 제한) | 오렌지 (`orange.50` + `orange.900`) |

## 규칙
1. 자동 닫힘: **3초** (에러는 5초)
2. 위치: 화면 하단 중앙, 바텀 내비 위 (`bottom: 80px`)
3. 동시 표시: **최대 1개** (새 토스트가 기존 토스트 대체)
4. 액션 버튼: 선택사항 (되돌리기, 보기, 재시도 등)
5. 텍스트: **1줄** 권장 (최대 2줄)
6. 진입 애니메이션: 아래에서 위로 슬라이드 (0.3s)

## 접근성
- Info/Success/Warning: `role="status"` + `aria-live="polite"`
- Error: `role="alert"` (즉시 읽힘)
- 스크린 리더가 자동으로 내용 읽음
- 액션 버튼은 키보드로 접근 가능
