# UI 상태 정의 가이드

> 모든 화면/컴포넌트는 아래 상태를 **반드시 고려**해야 합니다.

## 필수 상태 (Must-have)

### 1. Default
- 데이터가 정상적으로 로드된 기본 상태
- 모든 화면의 출발점

### 2. Loading
- 데이터를 불러오는 중
- **스켈레톤 UI** 사용 (전체 화면 스피너 금지)
- 기존 콘텐츠가 있다면 인라인 로딩 (하단 스피너)
- 최초 로딩과 추가 로딩을 구분

### 3. Empty
- 데이터가 0건인 상태
- 일러스트 + 안내 메시지 + CTA 버튼 조합
- 예: "아직 저장한 상품이 없어요" + [상품 보러 가기]

### 4. Error
- 네트워크 오류, 서버 에러 등
- 에러 메시지 + 재시도 버튼
- 예: "일시적인 오류가 발생했어요" + [다시 시도]
- HTTP 상태 코드나 기술적 메시지 노출 금지

### 5. Success
- 액션이 성공적으로 완료된 상태
- 토스트 또는 인라인 피드백
- 예: "저장되었습니다" (토스트, 3초 후 자동 닫힘)

### 6. Disabled
- 조건 미충족으로 인터랙션 불가
- `opacity: 0.4` + `pointer-events: none`
- 왜 비활성인지 **힌트 텍스트** 제공 권장

## 권장 상태 (Should-have)

### 7. Skeleton
- Loading의 하위 유형
- 실제 레이아웃과 동일한 형태의 회색 블록
- 애니메이션: 좌→우 shimmer 효과

### 8. Partial Error
- 화면 일부만 에러, 나머지는 정상
- 에러 영역만 별도 표시 (전체 에러 화면 X)

### 9. Offline
- 네트워크 연결 끊김
- 캐시된 데이터 표시 + 상단 배너 "오프라인 상태입니다"

### 10. Permission Denied (권한 없음)
- 접근 권한이 없는 화면/기능
- 안내 메시지 + 권한 요청 또는 대안 제시

### 11. First Time (온보딩)
- 사용자가 해당 기능을 처음 사용
- 코치마크, 툴팁 등으로 안내

### 12. Refreshing
- Pull-to-refresh 등 새로고침 중
- 기존 콘텐츠 유지 + 상단 로딩 인디케이터

## 컴포넌트별 상태 매트릭스

| 컴포넌트 | Default | Loading | Empty | Error | Disabled | Hover | Focus | Active |
|---|---|---|---|---|---|---|---|---|
| Button | O | O | - | - | O | O | O | O |
| Input | O | - | - | O | O | - | O | - |
| Card | O | O (Skeleton) | - | - | - | O | - | O |
| List | O | O | O | O | - | - | - | - |
| Modal | O | O | - | O | - | - | - | - |
| Tab | O | - | - | - | O | O | O | O |
| Toast | O (4종) | - | - | - | - | - | - | - |
| Navigation | O | - | - | - | - | - | - | O |

## 상태 전환 흐름

```
[진입] → Loading → Default
                 → Empty (데이터 0건)
                 → Error (로드 실패)

[액션] → Loading → Success (토스트)
                 → Error (인라인 에러)

[새로고침] → Refreshing → Default (갱신)
                        → Error (실패)
```

- [ ] Loading 상태 (스켈레톤 또는 스피너) 디자인 완료
- [ ] Empty 상태 (데이터 없음) 디자인 완료
- [ ] Error 상태 (에러 메시지 + 재시도) 디자인 완료
- [ ] Disabled 상태가 있는 경우 디자인 완료
- [ ] Success 피드백 (토스트/인라인) 정의 완료
- [ ] 권한 없음 케이스 고려 여부 확인
