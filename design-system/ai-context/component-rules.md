# 컴포넌트 사용 규칙 (Do's & Don'ts)

> AI 에이전트가 ODS 컴포넌트를 올바르게 사용하기 위한 규칙 모음

---

## 전역 규칙

### Do's
- CSS 변수(`var(--color-primary)`)로 색상 참조
- `font-family: var(--font-family)` 통일
- 간격은 4px 배수만 사용 (`--space-1` ~ `--space-8`)
- 인터랙티브 요소 최소 터치 타겟 44px
- `<button>`, `<a>`, `<input>` 등 네이티브 요소 우선
- 이미지에 `loading="lazy"` + 의미 있는 `alt`
- `box-sizing: border-box` 전역 적용

### Don'ts
- `!important` 사용 금지 (불가피한 경우 주석으로 사유 명시)
- `style=""` 인라인 스타일 금지
- 임의의 색상 HEX 직접 사용 금지 → 토큰 사용
- `div` + `onclick` 금지 → 시맨틱 `button` 사용
- `outline: none` 단독 사용 금지 → 커스텀 포커스 링 제공
- `user-select: none` 텍스트 콘텐츠에 사용 금지

## 컴포넌트별 규칙

### Button
| 규칙 | 설명 |
|---|---|
| Primary는 섹션당 1개 | 시각적 위계 유지 |
| 레이블은 동사형 | "저장하기" O / "저장" △ / "확인" O |
| 아이콘 버튼에 aria-label | 접근성 필수 |
| Loading 시 pointer-events: none | 중복 클릭 방지 |
| Danger는 확인 모달과 세트 | 실수 방지 |

### Card
| 규칙 | 설명 |
|---|---|
| 이미지 비율 통일 (4:3 or 1:1) | 그리드 정렬 |
| 타이틀 2줄 이내 | 넘으면 ellipsis |
| hover는 PC에서만 | `@media (hover: hover)` |
| Skeleton은 실제 레이아웃과 동일 형태 | UX 일관성 |

### Input
| 규칙 | 설명 |
|---|---|
| 라벨 필수 | 플레이스홀더는 라벨 대체 불가 |
| 에러 메시지는 구체적 | "올바르지 않습니다" X → "이메일 형식이 아닙니다" O |
| 모바일 font-size >= 16px | iOS 자동 확대 방지 |
| `aria-invalid` + `aria-describedby` | 에러 시 접근성 연결 |

### Navigation
| 규칙 | 설명 |
|---|---|
| Bottom Nav 탭 3~5개 | 5개 초과 금지 |
| 아이콘 + 라벨 조합 | 아이콘만 사용 금지 |
| `aria-current="page"` | 현재 페이지 표시 |
| Safe area 하단 패딩 | iPhone 노치 대응 |

### List
| 규칙 | 설명 |
|---|---|
| Empty에 CTA 포함 | "상품 보러 가기" 등 행동 유도 |
| Loading은 스켈레톤 | 전체 화면 스피너 금지 |
| 아이템 높이 최소 56px | 터치 타겟 확보 |
| Error에 재시도 버튼 | 사용자가 복구 가능하게 |

### Modal
| 규칙 | 설명 |
|---|---|
| 1 depth만 | 모달 위에 모달 금지 |
| ESC로 닫기 | 키보드 접근성 |
| 포커스 트랩 | Tab이 모달 내부에서만 순환 |
| 최대 너비 400px | 데스크톱에서도 집중 |

### Tab
| 규칙 | 설명 |
|---|---|
| 2~7개 탭 | 초과 시 가로 스크롤 |
| 활성 탭 1개만 | 동시 다중 선택 불가 |
| 라벨 2~4글자 | 간결하게 |
| 좌우 방향키 네비게이션 | 키보드 접근성 |

### Toast
| 규칙 | 설명 |
|---|---|
| 자동 닫힘 3초 (에러 5초) | 사용자 방해 최소화 |
| 동시 1개만 표시 | 새 토스트가 기존 대체 |
| 텍스트 1줄 권장 | 긴 내용은 모달 사용 |
| Error는 `role="alert"` | 스크린리더 즉시 읽기 |

### Chip
| 규칙 | 설명 |
|---|---|
| 높이 32px 고정 | 크기 통일 |
| 라벨 1~6글자 | 간결하게 |
| 멀티 선택 시 aria-pressed | 접근성 상태 표시 |
| 제거 버튼에 aria-label | "XX 필터 제거" |

### BottomSheet
| 규칙 | 설명 |
|---|---|
| 최대 85vh | 전체 화면 가리지 않음 |
| 바텀시트 안에 바텀시트 금지 | 1 depth |
| Handle bar 필수 | 드래그 인디케이터 |
| Danger 액션은 빨간 텍스트 | 시각적 구분 |
